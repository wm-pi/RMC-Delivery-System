# GOOD_EXAMPLES.md

이 문서는 프로젝트에서 반복해서 따라야 할 좋은 구현 패턴을 진행 중에 모으는 문서다.

프로젝트 시작 시 억지로 예시를 채우지 않는다. 실제 코드에서 좋은 패턴이 확인되면 링크와 이유를 추가한다.

---

## 작성 시점

- 같은 방식으로 재사용할 만한 route/service/repository 패턴이 생길 때
- 좋은 form, table, dialog, loading/error/empty state 구현이 생길 때
- 권한 처리, error handling, API client 사용이 모범적으로 정리된 코드가 생길 때
- AI가 따라 하면 좋은 파일 구조나 테스트 패턴이 생길 때
